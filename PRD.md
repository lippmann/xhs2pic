# 小红书长文转图片网站 PRD

版本：v0.1  
日期：2026-06-03  
阶段：需求讨论稿

## 1. 产品定位

做一个面向创作者的“小红书长文转图片”网页工具。用户把长文粘贴或上传到左侧编辑区后，可以继续修改文字、调整结构、插入图片；右侧实时预览按小红书推荐尺寸生成的多张图文卡片，每张卡片固定为 1080 x 1440 px、3:4 比例。

核心体验不是“能导出”，而是“文章放进去，立刻变成一组简洁、好看的小红书图文”。模板要克制、清爽、有编辑感，默认结果就应该能发。

## 2. 核心用户

1. 写长文的小红书创作者  
   需要把公众号、博客、笔记、书摘、课程稿快速转成图文。

2. 知识型内容运营  
   需要统一品牌样式，批量生产多页图文卡片。

3. 个人网站/工具站站长  
   需要一个可部署、可二次改造的静态或前端优先网站。

## 3. 核心目标

1. 输入长文后实时分页预览。
2. 每页预览严格为 1080 x 1440 px。
3. 用户可以看到总共会生成几张图。
4. 用户可以看到每张图截取到了哪些文字。
5. 用户可以调整主题、字体、行距、标题样式、字号、边距等关键参数。
6. 至少提供四套简洁好看的模板。
7. 用户可以在编辑区继续编辑文本，并添加图片。
8. 支持导出单张 PNG、全部 PNG、以及可选打包下载 ZIP。

## 4. 非目标

MVP 暂不做：

1. 登录、团队协作、云端草稿。
2. AI 改写、AI 配图、AI 摘要。
3. 后端队列式批量生成。
4. 小红书账号发布接口。
5. 复杂富文本编辑器，如 Notion 级块编辑、数据库、拖拽多栏布局。

这些可以作为后续版本。

## 5. 页面布局

整体为左右双栏工作台。

左侧：内容编辑与样式控制区。  
右侧：实时渲染预览区。

- 右栏主体展示当前选中卡片，旁边或底部显示页面缩略图列表。

移动端不强求左右同时展示，建议用顶部标签切换：

## 6. 内容输入与编辑能力

### 6.1 粘贴文章

用户可以直接复制粘贴文章到编辑区，这是最主要、最高频的入口。

支持纯文本、Markdown 基础语法：

- `#` 标题
- `##` 小标题
- 段落
- 引用
- 有序/无序列表
- 分隔线

粘贴后内容进入可编辑状态，用户可以继续修改文字、删除段落、补充标题或调整结构。

### 6.2 上传文章

MVP 支持：

- `.txt`
- `.md`
- `.docx`

后续可支持：

- URL 抓取
- 微信公众号文章解析

`.docx` 解析目标是提取文章内容结构，而不是复刻 Word 样式。第一版需要尽量保留：

- 标题/小标题
- 段落
- 有序/无序列表
- 加粗文本语义
- 引用或缩进段落的基础识别

暂不要求保留：

- Word 页眉页脚
- Word 原始字体和字号
- 复杂表格
- 批注、修订、脚注

### 6.3 编辑区能力

编辑区不是只读输入框，而是轻量内容编辑器。MVP 需要支持：

1. 修改正文文本。
2. 添加、删除、修改标题和小标题。
3. 添加、删除、修改段落。
4. 插入图片。
5. 删除图片。
6. 为图片添加可选说明文字。
7. 调整图片显示方式：铺满内容宽度、居中适配、圆角裁切。
8. 将选中文本设为标题、引用、列表或普通段落。

编辑区可以用 Markdown 编辑体验，也可以用所见即所得编辑器；但输出给分页系统的内容必须统一转换为结构化内容块。

### 6.4 图片能力

图片来源：

1. 用户从本地上传。
2. 用户直接粘贴剪贴板图片。
3. 用户拖拽图片到编辑区。

图片在预览中的规则：

1. 图片作为内容块参与分页。
2. 图片默认不跨页显示。
3. 如果当前页剩余空间不足，图片移动到下一页。
4. 如果单张图片过高，按模板规则缩放到当前页可容纳高度。
5. 图片说明文字跟随图片，尽量不与图片拆页。

MVP 暂不做复杂图片编辑，只提供基础显示方式和删除能力。后续可扩展裁剪、滤镜、贴纸和图片顺序拖拽。

### 6.5 内容解析

解析后生成结构化内容块：

- title
- subtitle
- paragraph
- quote
- list
- image
- imageCaption
- divider
- footnote

分页算法基于结构块，而不是简单按字符硬切。段落尽量不在中间断开；若单段过长，才允许按句子切分。

## 7. 实时预览与分页

### 7.1 画布尺寸

导出画布固定：

- 宽：1080 px
- 高：1440 px
- 比例：3:4

页面中按比例缩放展示，但 DOM/截图基准应保持 1080 x 1440。

### 7.2 分页规则

每页包含：

- 可选页眉：标题、系列名、页码。
- 正文区域：按模板设置留白。
- 可选页脚：作者名、日期、账号、页码。

系统需要实时计算：

- 总页数。
- 当前页包含的文字范围。
- 每页开头和结尾文本摘要。
- 溢出状态。

### 7.3 页面摘要

缩略图列表中每页显示：

```text
第 1 张
标题：为什么长文需要被重新排版
范围：开头 0 - 结束 426 字

第 2 张
范围：从“真正的问题不是内容太长...”到“读者需要呼吸感。”
```

## 9. 字体方案

中文开源字体建议：

1. Noto Serif SC：适合长文、书卷感、评论。
2. Noto Sans SC：现代、清晰、工具默认。
3. Source Han Serif SC / 思源宋体：正式、出版感。
4. Source Han Sans SC / 思源黑体：中性、信息密度高。
5. LXGW WenKai / 霞鹜文楷：柔和、有手写书卷感。
6. ZCOOL XiaoWei / 站酷小薇：适合标题，不建议大段正文。

英文字体建议：

1. Inter：现代 UI 与正文混排。
2. Lora：温和的 serif，适合引文。
3. Merriweather：长文阅读友好。
4. Source Serif 4：严肃编辑感。
5. EB Garamond：古典、文学感。
6. IBM Plex Sans：理性、技术感。

字体加载建议：

- 首选 Google Fonts / Bunny Fonts / 本地自托管三种策略可配置。
- 为中文字体提供系统 fallback。
- 导出前必须确保字体加载完成，否则截图会偏移。

## 13. 技术建议

推荐前端优先：

- React / Next.js 或 Vite + React。
- Zustand 管理编辑状态。
- Markdown 解析：remark / unified。
- 导出截图：html-to-image 或 dom-to-image-more。
- ZIP：JSZip。
- 字体加载：FontFace API + document.fonts.ready。

关键技术点：

1. 预览 DOM 按 1080 x 1440 真实尺寸渲染，再用 CSS transform 缩放。
2. 分页计算需要测量文本高度，建议用隐藏的 measurement canvas/DOM。
3. 导出前等待字体、图片、布局稳定。
4. 模板用 token 驱动，避免每套主题散落成独立逻辑。

## 14. 模板 token 结构

建议每套模板由配置驱动：

```ts
type TemplateTheme = {
  id: string
  name: string
  canvas: {
    width: 1080
    height: 1440
    background: string
    padding: { top: number; right: number; bottom: number; left: number }
  }
  typography: {
    cnFont: string
    enFont: string
    titleSize: number
    bodySize: number
    lineHeight: number
    paragraphGap: number
  }
  colors: {
    text: string
    muted: string
    accent: string
    rule: string
  }
  ornaments: {
    header: boolean
    footer: boolean
    texture: boolean
    pageNumber: "none" | "minimal" | "chapter"
  }
}
```

## 15. 用户流程

1. 用户打开网站。
2. 默认展示一篇示例文章和默认模板。
3. 用户直接粘贴长文，或上传 `.txt` / `.md` / `.docx` 文件。
4. 系统自动解析并进入可编辑状态。
5. 用户在左侧继续编辑文字、添加图片、调整标题或段落。
6. 用户调整模板、字体、行距、标题样式。
7. 右侧实时显示页数、每页文字范围、图片所在页和视觉效果。
8. 用户导出当前页或全部图片。

## 16. MVP 验收标准

1. 输入 3000 字中文长文，能稳定分页并显示总页数。
2. 每张导出 PNG 尺寸精确为 1080 x 1440 px。
3. 至少四套模板可切换。
4. 字体、字号、行距、边距调整会实时影响分页。
5. 每页缩略图能显示文本范围。
6. 用户可以直接粘贴文章，也可以上传 `.txt` / `.md` / `.docx`。
7. `.docx` 上传后能提取标题、段落、列表等核心内容。
8. 导入或粘贴后，用户可以继续编辑文本内容。
9. 用户可以上传或粘贴图片，图片作为内容块进入分页预览。
10. 导出前字体和图片加载完成，不出现布局变化。
11. 桌面端左右布局可用，移动端至少能切换输入和预览。

## 17. 第一版实现边界建议

第一版建议先做：

1. Vite + React 单页应用。
2. 纯前端生成，不依赖后端。
3. 支持直接粘贴文章。
4. 支持 `.txt` / `.md` / `.docx` 上传。
5. 支持轻量编辑区：改文字、改标题、插入/删除图片。
6. 四套模板。
7. PNG / ZIP 导出。
8. localStorage 保存最近一次内容和样式配置。

这样你可以部署到自己的任意静态网站、Vercel、Netlify、Cloudflare Pages 或传统服务器。

## 18. 已决策项

1. **默认主题**：Wenkai。
2. **封面图**：默认关闭。关闭状态下无独立封面页，第一页直接从标题 + 作者信息 + 摘要 + 正文开始排版。用户可手动开启封面图并上传图片。
3. **作者信息**：名字 + 头像存 localStorage，自动带入，用户可随时修改。头像由用户本地上传，转 base64 存储。
4. **字体加载**：自托管。字体文件部署在自己服务器，不依赖 Google Fonts，保证境内访问速度和截图导出时字体可靠性。
5. **图片导出**：右侧直接渲染分页预览，每页可单独下载 PNG，底部提供"下载全部 ZIP"按钮。
